# angst Refactor Plan

## Core Problems

| Problem | Impact |
|---|---|
| `hosts/` mixes system + user + domain selection | Git tracks machine state; conflicts on pull |
| `user.env` parsing duplicated in 7 files | Fragile, inconsistent |
| `x86_64-linux` hardcoded in 5 files | Not portable |
| `proj/angst` hardcoded in 10 files | Broke if cloned elsewhere |
| `allowUnfree` hardcoded in 3 files | No opt-out |
| `themes/default.nix` imported from lower layers | 3 layer violations |
| No standalone HM without host dir | Can't build for Debian/TTY |

## Solution: `config/local.nix`

Single gitignored source of truth for machine-specific state.

Each machine creates its own `config/local.nix`:

```nix
{
  host = "personal";
  username = "joao";
  theme = "github";
  monitors.primary = { name = "DP-1"; resolution = "1920x1080"; ... };

  domains = {
    shell.nushell.enable = true;
    terminal.zellij.enable = true;
    editor.nvim.enable = true;
    git.lazygit.enable = true;
    wm.i3.enable = true;
  };

  capabilities = {
    audio.enable = true;
    graphical.enable = true;
  };
}
```

## Phase 1 — Config layer

- Create `lib/env.nix` — central loader: tries `config/local.nix`, falls back to `user.env`, overlays `ANGST_*` env vars
- Create `config/local.nix.example` (tracked)
- Add `/config/local.nix` to `.gitignore`

## Phase 2 — Decouple hosts

- `hosts/<name>/default.nix` — strip `user`, `monitors`, `theme`, `repoPath`; keep only system platform
- `hosts/<name>/user.nix` — delete (moved to `config/local.nix`)
- `hosts/<name>/home.nix` — delete (domains in `config/local.nix`)
- `hosts/<name>/configuration.nix` — strip capabilities/theme enablement; keep hostname, hardware, kernel config
- `hosts/<name>/hardware.nix` — keep as-is (generated by nixos-generate-config)
- `common/home.nix` — delete or keep as default import only
- `common/user.nix` — delete
- `common/capabilities.nix` — keep as default NixOS module

## Phase 3 — Centralize env parsing

- `lib/parseEnv.nix` — keep for compat, add deprecation note
- `flake.nix` — replace inline parsing with `lib/env.nix`; derive `system` from nixpkgs
- `lib/flake/default.nix` — remove inline parsing; accept resolved config as param
- `lib/flake/homeConfigurations.nix` — simplify using resolved config
- `lib/build/mkHome.nix` — remove inline parsing; accept pre-resolved config
- `lib/build/mkHost.nix` — remove inline parsing; accept pre-resolved config

## Phase 4 — Fix hardcoded strings

- `x86_64-linux` → derive from `pkgs.system` or `builtins.currentSystem`
- `proj/angst` → derive from `self.sourceInfo` or config
- `allowUnfree` → move to `config/local.nix`

## Phase 5 — Fix architecture violations

- `capabilities/graphical.nix` → accept theme colors as parameter instead of importing themes
- `lib/build/mkHome.nix` → accept `themesLib` as parameter
- `lib/flake/default.nix` → accept `themesLib` as parameter

## Phase 6 — Flake outputs

```nix
{
  nixosConfigurations.default = ...;   # reads host from config/local.nix
  nixosConfigurations.personal = ...;
  nixosConfigurations.generic = ...;
  nixosConfigurations.ssh = ...;

  homeConfigurations.default = ...;    # reads everything from config/local.nix
  homeConfigurations."joao@desktop" = ...;  # explicit attrpaths still work
}
```

## Usage

| Machine | Config | Command |
|---|---|---|
| NixOS desktop | `config/local.nix` with host + domains | `nixos-rebuild switch --flake .#default` |
| Debian TTY server | `config/local.nix` with username + tty domains | `home-manager -f .#homeConfigurations.default` |
| Debian JS/PHP server | `config/local.nix` with username + toolchain domains | `home-manager -f .#homeConfigurations.default` |
